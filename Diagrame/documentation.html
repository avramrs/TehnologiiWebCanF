Abstract

These days keeping track of information is a very important issue. Stocks management is a part of this issue, companies can benefit from a way to keep track of the information about their stocks. An interface is needed to ease the job of users and handle this task.

Short Description

CanF is a web application that is used for managing information about food kept in cans, boxes or any other form of packaging. The goal of our project is to provide a way to manage food so that users can easily and efficiently find any information they may need about it. Users can upload informations about cans and download them at the push of a button.

GitHub

Github was used for version control during development. Different branches were used for developing individual pages and functionalities. This approach was chosen so that pages could be created without disrupting the work of the others. Moreover it allowed each of us to test the functionality localy before integration.

Database

For the purpose of managing our data we are using a relational database. MySQL was chosen for it's ease of use. Two tables were created, one to keep track of our users and the other to manage the informations about our products. The users table contains the id, username and the encrypted hash value of the password. These are created after registering, each username and id being unique. The products table contains details about the food and metadata. The only required fields in this table are the ids and the name of the product.

Usecase

The user can access the application only if he is logged in. The first page he sees is the Login page. Here the user can either login with an existing account or register with a new account using a username and a password. The logout button can be used to return here. The next page after logging in is the Products page. Here he can see all the products that were uploaded so far and a few buttons. A product can be clicked in order to see informations about it. He can see here by default 2 buttons that are used for downloading the said informations. If the user is an administrator or the owner (the user that uploaded the information) he can also delete the product or update it's information.

  • The stocks button downloads a pdf that can be used to see all the latest products that are in stock.
  • The Import button redirects the user to an upload page where he upload information about cans using different file formats.
  • The Contacts button shows the page that gives ways to contact us.

    • Implementation

    The main tools were php, html, mysql and ajax. The login page checks if the username and password hash are the same as the one in the database. In the register page the input is checked so that the password is bigger than 6 characters and the user doesn't already exist. Each page checks if the user is logged in using sessions, if not he is reddirected to the login page. The products page uses ajax to update the page in real time so that the user can see if cans are updated. The stocks button queries the database and uses fpdf to create a pdf with the data. The import part uses simpleXML tools and other functions to check if the xml or csv is valid and if it respects the required format having at least the name of the product. It sanitizez the input and then uploads it to the database. In the page of a product... The user can also choose to upload information if he has the same id as the user id in the database or if he is the admin. He is redirected to a form where he can change the fields that are already completed with the data from the database. The form checks if the user is admin or the owner again for security reasons and it also checks the input for code injection.
